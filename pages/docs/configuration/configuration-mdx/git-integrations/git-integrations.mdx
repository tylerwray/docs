# Git Integrations

The following configuration options can be used through a `now.json` file like the [options above](#configuration).

## github.enabled

**Type**: `Boolean`.

When set to `false`, [Now for GitHub](/docs/v2/advanced/now-for-github) will not deploy the given project regardless of the GitHub app being installed.

<Example>
  <b>Usage Example:</b>
  <Code lang="json">{`{
  "github": {
    "enabled": false
  }
}`}</Code>
</Example>

## github.autoAlias

**Type**: `Boolean`.

When set to `false`, [Now for GitHub](/docs/v2/advanced/now-for-github) will not apply the alias upon merge.

<Example>
  <b>Usage Example:</b>
  <Code lang="json">{`{
  "alias": ["my-zeit-website.com"],
  "github": {
    "autoAlias": false
  }
}`}</Code>
</Example>

## github.silent

**Type**: `Boolean`.

When set to `true`, [Now for GitHub](/docs/v2/advanced/now-for-github) will stop commenting on pull requests and commits.

<Example>
  <b>Usage Example:</b>
  <Code lang="json">{`{
  "github": {
    "silent": true
  }
}`}</Code>
</Example>

## github.autoJobCancelation

**Type**: `Boolean`.

When set to false, [Now for GitHub](/docs/v2/advanced/now-for-github) will always build pushes in sequence without cancelling a build for the most recent commit.

<Example>
  <b>Usage Example:</b>
  <Code lang="json">{`{
  "github": {
    "autoJobCancelation": false
  }
}`}</Code>
</Example>
